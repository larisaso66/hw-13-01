`Обнаруженные уязвимости машины Metasploitable`

1. **Уязвимость в FTP-сервере vsftpd (порт 21)**

   **Версия службы:** vsftpd 2.3.4

   **Уязвимость**: Эта версия FTP-сервера содержит бэкдор, внедренный злоумышленниками в исходный код. Если в имени пользователя отправить специально заданные символы ":)", на порту 6200 открывается shell с правами root. Это позволяет полностью скомпрометировать систему без аутентификации.

   **CVE идентификатор**: CVE-2011-2523

   **Ссылка на Exploit-DB**: https://www.exploit-db.com/exploits/49757
   
2. **Уязвимость в IRC-сервере UnrealIRCd (порт 6667)**

   **Версия службы**: UnrealIRCd (версия 3.2.8.1)

   **Уязвимость**: В дистрибутив UnrealIRCd версии 3.2.8.1 был подменен архив с исходным кодом. В него добавили троян, который позволяет выполнить любую команду на сервере, отправив специально сформированный пакет.

   **CVE идентификатор**: CVE-2010-2075

   **Ссылка на Exploit-DB**: https://www.exploit-db.com/exploits/16922
   
3. **Уязвимость в Samba (порты 139, 445)**

   **Версия службы**: Samba smbd 3.X - 4.X (версия 3.0.20)

   **Уязвимость**: В Samba версии 3.0.20 и более ранних существует уязвимость, связанная с ошибкой в функции chain_reply. Она позволяет удаленно вызвать повреждение памяти и выполнить вредоносный код с правами root.

   **CVE идентификатор**: CVE-2010-2063

   **Ссылка на Exploit-DB**: https://www.exploit-db.com/exploits/16320

**1. Чем отличаются эти режимы сканирования с точки зрения сетевого трафика?**

Главное отличие заключается в отправляемых пакетах и используемых протоколах:

- **SYN Scan** отправляет TCP-пакеты только с флагом SYN;

- **FIN Scan** отправляет TCP-пакеты только с флагом FIN, который обычно используется для завершения соединения;
  
- **Xmas Scan** отправляет TCP-пакеты с одновременно установленными флагами FIN, PSH и URG;
  
- **UDP Scan** отличается тем, что вместо TCP-сегментов сканер отправляет UDP-дейтаграммы (как пустые, так и с полезной нагрузкой) на целевые порты.

**2. Как отвечает сервер?**

Ответ сервера зависит от состояния порта и типа полученного пакета:

- **При SYN Scan**: на открытые порты сервер отвечает стандартным SYN-ACK, приглашая к диалогу. На закрытые порты приходит RST-ACK, обрывая попытку соединения;

- **При FIN и Xmas Scan**: на открытые порты ответа нет (пакет игнорируется). На закрытые порты приходит RST-ACK.

- **При UDP Scan**: на закрытые UDP-порты сервер отправляет ICMP-пакет с сообщением "Destination Unreachable (Port unreachable)". Открытые порты либо не отвечают (что трактуется как open\ или filtered), либо могут прислать ответ от приложения.
